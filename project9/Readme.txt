# Проект 9. CAD-система для сварных узлов

# Описание проекта

Проект представляет собой систему автоматизированного проектирования (CAD) для моделирования сварных конструкций. Реализованы три типа сварных соединений (примитивов): стыковой шов (ButtWeld), угловой шов (FilletWeld) и тавровый шов (EdgeWeld). Каждый примитив вычисляет объём наплавки, массу наплавленного металла и прочность соединения.

Сборка (Assembly) объединяет несколько примитивов, автоматически пересчитывает суммарные характеристики и уведомляет наблюдателей об изменениях. Паттерн "Наблюдатель" реализован для автоматического обновления чертежей, отчётов по массе и проверки прочности.

Проект включает:
- Генератор случайных сварных конструкций (100+ примитивов)
- Сохранение данных в CSV и JSON
- Визуализацию распределения масс и зависимости прочности от длины
- Логирование всех значимых событий
- Модульные тесты (pytest)
- Пользовательские исключения

# Структура проекта

├── main.py # Точка входа: демонстрация работы, запуск генерации и визуализации
├── test_primitives.py # Модульные тесты для примитивов
├── test_assembly.py # Модульные тесты для сборки и наблюдателей
└── README.md # Этот файл

Запустите главный скрипт:
python main.py

Что произойдёт:

-Создастся сборка "MainFrame".

-Добавятся три примитива (стыковой, угловой, тавровый), затем удалится один из них.

-Будут уведомлены наблюдатели (чертёжник, отчёт по массе, проверка прочности).

-Выведутся основные характеристики сборки.

-Сгенерируются 10 сборок, в каждой по 10 случайных примитивов (всего 100 примитивов).

-Данные сохранятся в папку project9/assemblies.csv и project9/assemblies.json.

-Построятся два графика: распределение масс сборок (горизонтальная гистограмма) и зависимость прочности от длины примитивов (точечный график с линией тренда).

-Графики сохранятся в project9/mass_distribution.png и project9/strength_vs_length.png.

-Логи сохранятся в project9/cad_system.log.


Кирилл: создаёт абстрактный класс WeldPrimitive. Наследники ButtWeld, FilletWeld, EdgeWeld реализуют расчёты по своим формулам. super() для общих атрибутов (длина, материал).
Никита: разрабатывает класс Assembly, содержащий список примитивов (композиция). Методы add_primitive(), remove_primitive(), calculate_total_mass(). Observer: при изменении сборки уведомляет DrawingUpdater, MassReporter, StrengthChecker. Генерация тестовых сборок, обработка исключений (несовместимые примитивы, нулевая длина) и построение гистограмм.
